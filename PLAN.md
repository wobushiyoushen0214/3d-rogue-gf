# 3D IT 职业成长肉鸽统一计划（替换国风题材）

## Summary
- 项目定位调整为：`单人爽刷成长` 的 `IT 职业题材 3D 肉鸽`。
- 核心体验保持不变：`低门槛移动操作 + 自动攻击 + 高频成长反馈 + 流派成型 + 长线重复开局`。
- 题材包装从“国风修真”切换为“互联网研发团队攻坚”：敌人是 `Bug 潮`、`需求轰炸`、`线上事故`、`技术债`。
- 当前项目继续沿用现有 3D 战斗模板与目录结构，不推翻底层技术，仅替换玩法语义、内容配置、职业与技能树。

## 执行偏差自检（自我分析）
- 问题 1：上版计划偏“方向描述”，缺少可直接开发的 `任务拆分 + 验收标准 + 执行顺序`。
- 问题 2：职业与技能树已经定义，但 `敌人生态` 还不够具体，缺少有记忆点的精英与 Boss。
- 问题 3：没有明确“先做什么、做到什么算完成”，导致执行阶段容易漂移。
- 纠偏原则：
  - 计划必须落到 `文件级改造点` 与 `当周可交付项`。
  - 每个阶段必须包含 `可测结果`，不是只写愿景。
  - 先打通 `学生 -> 专职 -> 敌潮压制 -> 单局结算` 这条最短闭环，再扩内容。

## 产品方向
### 1. 题材与定位
- 主题改为 `IT 职业成长模拟 + 生存战斗`，不再使用国风修真包装。
- 战斗表现改为：`代码提交`、`接口联调`、`需求拆解`、`排期推进`、`缺陷治理`、`上线应急`。
- 美术与 UI 风格建议：`赛博办公室 / 数据可视化 / 工位与机房`，突出“技术攻防”的职业感。

### 2. 核心体验
- 输入层面：玩家主要控制 `走位与选位`，攻击与大部分技能 `自动释放`。
- 成长层面：通过 `处理任务 -> 获得经验 -> 升级专精 -> 构建技术流派 -> 对抗更高压项目波次`。
- 反馈层面：强调 `问题被解决`、`指标上升`、`日志清屏`、`连锁优化` 带来的持续正反馈。
- 长线层面：后续扩展 `章节项目`、`无尽冲刺`、`局外成长`、`职业专职`、`团队组合`。

## 角色系统（核心改造）
### 1. 初始职业
- 所有玩家开局统一为：`计算机专业学生`。
- 学生阶段关键词：`基础知识广但深度不足`，适合过渡到任一专职。
- 学生初始能力示例：
  - `数据结构基础`：基础攻击 +5%
  - `计算机网络`：技能冷却 -5%
  - `操作系统`：受控时间 -10%

### 2. 专职解锁（建议 Lv3 或第 1 章结算后）
玩家可从以下 6 个角色中选择专职：
- `前端工程师`
- `后端工程师`
- `产品经理`
- `项目经理`
- `测试工程师`
- `实施顾问`

### 3. 角色定位与真实技术栈
| 角色 | 定位 | 真实技术栈（示例） | 核心成长属性 |
|---|---|---|---|
| 前端工程师 | 高频输出 / 范围清怪 | `JavaScript` `TypeScript` `Vue` `React` `Vite` `Webpack` | 攻速、范围、连击 |
| 后端工程师 | 稳定持续输出 / 单体爆发 | `Java` `Spring Boot` `MySQL` `Redis` `Kafka/RabbitMQ` `Docker` | 攻击、穿透、续航 |
| 产品经理 | 控场 / 增益 / 路线规划 | `PRD` `Figma` `Axure` `A/B Test` `SQL基础` `用户研究` | 控制、增益、资源效率 |
| 项目经理 | 团队增幅 / 节奏管理 | `Scrum` `Kanban` `Jira/TAPD` `风险管理` `里程碑管理` | 冷却缩减、协同、容错 |
| 测试工程师 | 弱点打击 / Debuff / 反制 | `Python` `Selenium` `Cypress` `JMeter` `Postman` `CI/CD` | 暴击、减抗、异常处理 |
| 实施顾问 | 生存 / 支援 / 场景适配 | `ERP/CRM` `SQL脚本` `Linux部署` `网络配置` `API集成` | 防御、恢复、适配性 |

## 技术树设计（每个职业独立）
> 说明：每个职业提供 3 条主分支，玩家局内通过“升级三选一”构筑 build。

### 1. 前端工程师技术树
- `组件化分支`：Vue 组件复用、状态管理、微前端
- `工程化分支`：Vite 构建加速、CI 静态检查、自动化发布
- `性能优化分支`：首屏优化、懒加载、虚拟列表
- 玩法效果倾向：`多段弹幕`、`范围覆盖`、`刷新快`

### 2. 后端工程师技术树
- `服务架构分支`：Spring Boot 微服务、接口治理、服务发现
- `数据链路分支`：MySQL 索引、Redis 缓存、读写分离
- `高并发分支`：消息队列削峰、限流降级、熔断重试
- 玩法效果倾向：`单发高伤`、`持续 DoT`、`Boss 战强`

### 3. 产品经理技术树
- `需求洞察分支`：用户画像、场景拆解、价值优先级
- `方案设计分支`：PRD 完整度、交互原型、验收标准
- `增长实验分支`：A/B 策略、漏斗优化、留存机制
- 玩法效果倾向：`全局增益`、`目标标记`、`路线控制`

### 4. 项目经理技术树
- `计划排期分支`：迭代规划、里程碑压缩、关键路径优化
- `风险管理分支`：风险预警、故障预案、资源兜底
- `协作机制分支`：跨团队同步、会议效率、阻塞清除
- 玩法效果倾向：`团队 buff`、`冷却缩短`、`容错提升`

### 5. 测试工程师技术树
- `自动化分支`：UI 自动化、接口自动化、回归流水线
- `性能分支`：压测建模、瓶颈定位、稳定性回归
- `质量防线分支`：缺陷预测、冒烟策略、质量门禁
- 玩法效果倾向：`弱点暴击`、`减抗破甲`、`异常反制`

### 6. 实施顾问技术树
- `部署交付分支`：环境搭建、发布脚本、版本回滚
- `业务适配分支`：流程配置、权限模型、主数据治理
- `现场支持分支`：故障定位、培训赋能、跨系统集成
- 玩法效果倾向：`护盾`、`持续回复`、`应急救场`

## 完整玩法循环（IT 语义）
### 1. 长循环
- 玩家从 `主界面（人才市场）` 进入战斗。
- 开局配置：`职业`、`初始技能包`、`工具链`、`个人天赋`。
- 局内通过 `走位生存 + 自动输出 + 升级三选一` 持续成长。
- 单局结束后结算 `经验 / 资源 / 专业认证点`，用于局外成长。

### 2. 单局循环
- `0-60 秒`：新人任务（低压）
- `1-3 分钟`：需求波次上升，开始形成职业流派
- `3-6 分钟`：复合问题增多，考验角色独特机制
- `6 分钟后`：高压阶段，触发 `线上事故` 或 `项目 Boss`

## 成长与难度系统（新增扩展）
### 1. 等级节奏（前期快、后期深）
- `Lv1 -> Lv2`：目标体感为击杀 `1-3` 只普通怪即可升级（当前按 2 点经验设计）。
- `Lv2-Lv3`：仍保持快速成型，帮助玩家在 1 分钟内进入“有 build 的状态”。
- `Lv4+`：经验需求采用递增曲线，强调“越往后越需要路线和操作配合”。
- 设计原则：前期避免劝退，中后期拉开强度与 build 差异。

### 2. 无限等级成长
- 取消等级上限，改为 `无限等级` 成长，后期通过递增经验成本自然拉长节奏。
- 高等级阶段的目标不再是“是否满级”，而是“build 完成度 + 生存时长 + 压力处理效率”。

### 3. 升级奖励与技能点
- 每次升级仍保留 `三选一强化`，用于即时属性/机制成长。
- 新增 `技能点` 里程碑：每 `5` 级获得 `1` 点技能点。
- 技能点定位：用于后续分支里程碑强化、主动技能插槽或职业大招解锁。

### 4. 怪物动态难度（时间 + 关卡 + 等级）
- 刷怪间隔随 `战斗时长` 与 `玩家等级` 双轴降低，怪潮越来越密。
- 每波刷怪数量随时间和等级同步提升，后期怪压更明显。
- 怪物基础血攻按动态系数上升，确保“高等级玩家 + 长时间存活”仍有挑战。
- Boss 与精英沿用既有机制，同时吃到动态难度的基础加成。

### 5. 该模块验收标准
- 开局 30-60 秒内能稳定升到 2-3 级，不会出现“前期升级过慢”。
- 每 5 级必定获得技能点，并有明确 UI/提示反馈。
- 5 分钟后怪潮明显更快、更硬；8-10 分钟后压力峰值稳定出现。
- 无限等级下不出现经验溢出、等级卡死、升级面板异常等阻断问题。

## 敌人与事件设计（追加）
### 1. 基础敌人
- `需求怪`：移动快、血量低、数量多，压走位。
- `Bug 怪`：移动慢、血量中、会远程抛错，压续航。

### 2. 精英敌人
- `代码屎山`（精英）：
  - 机制：高护甲、低移速、接触后给玩家施加 `维护负担`（攻速下降）。
  - 特性：被击破时分裂出 2-3 个 `历史遗留 Bug` 小怪。
  - 设计意图：迫使玩家选择“先处理屎山”还是“继续清小怪”。

### 3. Boss 事件
- `老板的大饼`（Boss 事件）：
  - 机制 1：周期释放 `愿景冲刺`，临时提高全场敌人移速。
  - 机制 2：释放 `资源画饼`，场上出现虚假增益点，触碰后短暂降低输出。
  - 机制 3：血量降至 30% 后进入 `再坚持一下` 阶段，召唤两波需求怪。
  - 设计意图：通过“高压承诺+资源错配”制造节奏波峰，形成关卡记忆点。

## 升级计划（V2 执行版）
### 1. 里程碑拆分
- `M1（底盘可玩）`：学生职业 + 基础刷怪 + 升级三选一 + 结算。

- `M8（版本验收与重构）`：做一轮 10 分钟稳定性测试、数值回收、坏味道代码整理。

### 1.5 计划维护机制（新增强制要求）
- 每完成一个阶段，必须立即回写 `PLAN.md`，不能只改代码不改计划。
- 每次回写计划时，必须同时做 3 件事：
  - `阶段复盘`：本阶段实际完成了什么，和原计划相比哪里超前/缺失。
  - `计划扩充`：把下一阶段拆到文件级、系统级、验收级，不允许只写一句“继续优化”。
  - `风险刷新`：补充新发现的 Bug、体验问题、结构债务、待验证项。
- 每次扩充计划后，`执行进度` 必须追加具体条目，不覆盖旧记录，保留演进轨迹。
- 每一轮新增内容至少覆盖：
  - `玩法`
  - `UI/HUD`
  - `数值/平衡`
  - `测试/验收`
- 若某阶段出现偏差，先在计划中写 `自我分析 + 纠偏动作`，再继续执行。
