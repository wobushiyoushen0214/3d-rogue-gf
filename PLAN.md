# 3D IT 职业成长肉鸽滚动计划

## 1. 项目定位
- 项目主题已经从国风题材切换为 `IT 职业成长 + 生存肉鸽`。
- 玩家开局身份为 `计算机专业学生`，局内成长后可专职为 `前端 / 后端 / 产品 / 项目 / 测试 / 实施`。
- 核心体验保持为 `移动走位 + 自动攻击 + 高频升级反馈 + 技术树 build 成型 + 长局压力递增`。
- 世界语义统一为互联网团队攻坚：敌人是 `需求怪 / Bug 怪 / 代码屎山 / 老板的大饼`，玩家输出是 `技术动作 / 工具链 / 工程能力`。

## 5. 自我分析与纠偏
- 问题 1：之前的计划累积了太多阶段日志，开发时很难快速看出 `什么已经真做了`、`什么只是规划`。
- 问题 2：战斗主链路推进很快，但 `实机回归`、`HUD 可读性`、`文本统一` 没有同步跟上。
- 问题 3：技能点和技术树已经有框架，但如果不尽快做成强反馈节点，玩家会只觉得是在堆数字。
- 纠偏原则：
  - 以后 `PLAN.md` 只保留一版主计划，阶段记录写成简洁滚动更新，不再重复堆同类段落。
  - 每做完一轮代码，必须同步写清 `已完成 / 进行中 / 未完成`。
  - 优先把 `可玩闭环 + 实机回归 + 数值手感` 做扎实，再继续扩系统层数。

## 6. 下一阶段开发计划

### 阶段 G1：实机回归与问题清单
- 优先级：`P0`
- 目标：在 Cocos 编辑器内完整跑通一局 8-10 分钟。
- 重点验证：
  - 首页 -> 选职 -> 开局 是否稳定。
  - 普通怪、精英、Boss 是否都能正常生成、受伤、死亡。
  - 升级、转职、技能点、突破是否都能在同一局共存。
  - 玩家死亡后是否一定能进入 GameOver，并且支持重开。
- 验收标准：
  - 无阻断报错。
  - 无“不能动 / 不刷怪 / 不发子弹 / 不掉血 / 不会死”这类阻断问题。

### 阶段 G2：数值节奏校准
- 优先级：`P0`
- 目标：把当前体验调整到明确的 `前快后难`。
- 核心标准：
  - `Lv.1 -> Lv.2` 体感保持在击杀 `1-3` 只普通怪。
  - 开局普通怪保持 `一枪可清`。
  - `3-5 分钟` 明显进入 build 成型期。
  - `8-10 分钟` 出现稳定压力峰值，但不是纯数值碾死玩家。
- 主要文件：`assets/script/view/game/PlayerTs.ts` `assets/script/view/game/level.ts` `assets/script/const/LevelConfig.ts`

### 阶段 G3：HUD 和文案清理
- 优先级：`P1`
- 目标：把局内反馈从“能看懂”升级到“看一眼就知道当前 build 状态”。
- 内容：
  - 技能点突破成功增加更强提示。
  - 主修分支高亮更清楚。
  - 统一运行时提示的语言风格，尽量改成中文 IT 表达。
  - 清理历史乱码注释和明显脏文案。
- 主要文件：`assets/script/view/game/UIGameTs.ts` `assets/script/view/game/UIMaterInfoTS.ts` `assets/script/view/game/PlayerTs.ts`

### 阶段 H1：技能点系统深化
- 优先级：`P1`
- 目标：让技能点不只服务被动突破，而是进入更高层成长节点。
- 方向：
  - 主动技能槽原型。
  - 分支终极节点。
  - 职业专属大招或短时爆发。
- 验收标准：
  - 至少 1 个职业能通过技能点路线拿到显著区别于普通升级的能力。

### 阶段 G4：中局事件扩充
- 优先级：`P1`
- 目标：补一层 `2-6 分钟` 的中局事件，让单局节奏不只依赖普通刷怪和 Boss 倒计时。
- 方向：
  - `需求轰炸`：短时间追加波次，制造清怪压力峰值。`已完成`
  - `排期冲刺`：阶段性提高刷怪频率，并让怪潮更贴脸。`已完成`
  - `项目评审`：生成更克制走位或输出节奏的特殊波次。`已完成`
  - `线上事故`：制造高移速或高压迫的突发事件波次。`已完成`
  - `特殊波次`：继续扩展护盾怪、冲锋怪、投射怪等主题小波次。`下一步`
  - `事件预警`：加入 HUD 提示，让玩家知道当前压力来源。`已完成第一版`
- 验收标准：
  - 中局至少出现 4 类可感知事件。
  - 玩家能通过提示区分“普通刷怪”与“事件波次”。
  - 不依赖新增美术资源也能稳定跑通。

### 阶段 H2：外层产品化
- 优先级：`P2`
- 目标：补齐从“能玩”到“像个完整游戏”的外层内容。
- 内容：
  - 首页视觉强化。
  - 结算页 / 评分页 / 重开流程。
  - 局外成长原型。
  - 章节主题包装。
